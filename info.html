¿Qué es un sistema operativo?

Software principal de todo dispositivo electrónico que administra todo el entorno siendo intermediario entre el hardware, aplicaciones y usuario.

Funciones y objetivos del sistema operativo

El sistema operativo es el único programa que interactúa directamente con el hardware de la computadora.
Sus funciones primarias son:

-Abstracción

Los programas no deben tener que preocuparse de los detalles de acceso a hardware, o de la configuración particular de una computadora. El sistema operativo se encarga de proporcionar una serie de abstracciones para que los programadores puedan enfocarse en resolver las necesidades particulares de sus usuarios.
Un ejemplo de tales abstracciones es que la información está organizada en archivos y directorios (en uno o muchos dispositivos de almacenamiento).
Memoria, espacio de almacenamiento, tiempo de procesamiento, etc. Los diferentes procesos que se ejecuten en él compiten por ellos. el sistema operativo puede implementar políticas que los asignen de forma efectiva y acorde a las necesidades establecidas para dicho sistema.

-Aislamiento

En un sistema multiusuario y multitarea cada proceso y cada usuario no tendrá que preocuparse por otros que estén usando el mismo sistema.

-Proceso por lotes (batch processing)

En los años cincuenta aparecieron los dispositivos perforadores/lectores de tarjetas de papel, los programadores entregaban su lote de tarjetas perforadas (en inglés, batches) a los operadores, quienes las alimentaban a los dispositivos lectores, que lo cargaban en memoria en un tiempo razonable, iniciaban y monitoreaban la ejecución, y producían los resultados.

Clasificación de los sistemas operativos

-Sistemas operativos monousuarios

Soportan a un usuario a la vez, sin importar el número de procesadores que tenga la computadora o el número de procesos o tareas que el usuario pueda ejecutar en un mismo instante de tiempo (computadoras personales).

-Sistemas operativos multiusuarios

Dan servicio a más de un usuario a la vez, ya sea por medio de varias

-Sistemas operativos mono tareas

Sólo permiten una tarea a la vez por usuario.

-Sistemas operativos multitareas

Un sistema operativo multitarea es aquél que le permite al usuario estar realizando varias labores al mismo tiempo.

-Sistemas operativos uniproceso

Un sistema operativo uniproceso es aquél capaz de manejar solamente un procesador de la computadora, de manera que si la computadora tuviese más de uno le sería inútil. El ejemplo más típico de este tipo de sistemas es el DOS.

Sistemas operativos multiproceso

Un sistema operativo multiproceso se refiere al número de procesadores del sistema, que es más de uno y éste es capaz de usarlos todos para distribuir su carga de trabajo.

Modelos de Sistemas Operativos

-Modelo para un solo usuario

-Modelo para una red

Un sistema operativo personal o de usuario único, donde un individuo utiliza una computadora o dispositivos para sus propias tareas y actividades. Proporciona herramientas para administrar archivos, ejecutar aplicaciones y realizar tareas en la computadora de manera cómoda y segura.

Estructura de un modelo para un solo usuario

Kernel del sistema operativo

  • Es el corazón del sistema operativo
  • Administra los recursos del hardware, como memoria y CPU
  • Proporciona una interfaz para que las aplicaciones y el usuario interactúen con el sistema.

Interfaz de usuario

  • Ofrece una interfaz de usuario amigable para que el usuario interactúe con el sistema.
  • Esto puede ser una interfaz gráfica de usuario (GUI)
  • Una interfaz de línea de comandos (CLI)

Gestión de Archivos

El S.O maneja la organización y el acceso a los archivos y directorios. Los usuarios pueden crear, abrir, modificar, mover y eliminar archivos utilizando comandos o una interfaz gráfica.

Gestión de Procesos

El sistema operativo administra los procesos en ejecución. Puede haber múltiples aplicaciones en ejecución al mismo tiempo. El S.O se encarga de programar y asignar tiempo de SPW a cada proceso.

Gestión de Memoria

Administra la memoria del sistema, asegurando que las aplicaciones tengan suficiente espacio para ejecutarse. Esto puede incluir administración de memoria virtual y técnicas de paginación.

Gestión de dispositivos

El sistema operativo maneja la comunicación entre las aplicaciones y los dispositivos periféricos E/S.

Seguridad y autenticación

Proporciona mecanismos de seguridad para proteger los datos del usuario y su privacidad. (Contraseñas, sistemas de archivos encriptados y cortafuegos básicos).

Redes

Proporciona capacidades de red para permitir que el usuario se conecte a internet y otros dispositivos en red.

Actualizaciones y Mantenimiento

Puede proporcionar actualizaciones automáticas y herramientas de mantenimiento para asegurarse de que el sistema esté actualizado y funcionando correctamente.

Aplicaciones

Puede incluir aplicaciones preinstaladas o proporcionar una tienda de aplicaciones donde el usuario puede descargar e instalar programas adicionales.

Memoria

Asignación de espacio contiguo

  • Asignación dinámica de memoria particionada
  • Protección y uso compartido

Asignación de espacio no contiguo

  • Segmentación
  • Paginación
  • Segmentación con paginación

Gestión de memoria virtual

  • Introducción
  • Paginación por demanda
  • Algoritmos de reemplazo de páginas
  • Algoritmos de asignación de marcos de página

ANTECEDENTES

  • Un programa debe cargarse en memoria desde disco y colocarse dentro de un proceso para que se ejecute
  • La memoria principal y los registros son los únicos dispositivos de almacenamiento a los que puede acceder la CPU directamente
  • El acceso a registro es muy rápido; supone un ciclo de CPU (o menos)
  • El acceso a memoria principal puede durar varios ciclos
  • Las memorias caché se colocan entre la memoria principal y la CPU para acelerar el acceso a la información
